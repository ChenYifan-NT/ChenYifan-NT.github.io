LeetCode 题型、算法和技巧总结

Posted on 2019-11-01 Edited on 2019-11-03 In LeetCode

这里总结了刷题过程中遇到的题型、算法和技巧。遇到只靠文字难以理解的部分，可以点击 tag 找到相关例题。各种题型和算法也可以通过 tag 找到对应的例题。

技巧

循环数组

循环数组指的是头尾相接的数组。解决这类问题有三种方法，分别是分裂、重复和取反。

分裂法根据题目要求，将原数组分裂成多个数组分别进行研究。例如有些题目，数组头尾的元素不能同时被选择，则可以生成一个只含头元素和一个只含尾元素的数组分别解决。

滚动指针

某些动态规划问题中需要用前几个状态推导后一个状态。这时候不需要记录每个点的状态，而只需要记录与推导相关的几个点的状态。更进一步，如果最终状态会覆盖前几个状态中的一个，只需要一个位置去记录这两个状态就足够了。

滚动数组

在某些二维动态规划问题中，下一行的状态需要用上一行的数据进行推导。这时候不能再使用滚动指针，而要改成滚动数组。无论是滚动指针还是滚动数组，都可以在非滚动的基础上增加模运算来实现。使用滚动数组的时候时刻要注意，当前正在更新的行中，是否有些位置的数据需要进行重置。

事实上我的经验是，用滚动指针和滚动数组反而会降低成绩，因为模运算也要消耗时间。同时在有限的数据规模下，提升的空间性能也是十分有限的

方形坐标

对于一个二维矩阵中的长方形，只需要四个变量（lx, ly, rx, ry）就可以进行定位。如果是正方形，则只需要三个变量（x, y, a）就可以进行定位。

方向数组

在一些二维数组题目中有时需要向临近坐标遍历。这时候先定义一个方向数组就能让遍历变得十分简单。方向数组本身不复杂，这里就不详细记录了。

动态规划

动态规划四要素

  1. 状态（State）
    • 最优解/Maximum/Minimum
    • Yes/No
    • Count(*)
  2. 状态转移方程（Function）
    • 状态之间的联系
  3. 初始化（Initialization）
    • 初始状态
  4. 答案（Answer）
    • 最终状态

动态规划分类

动态规划可以分为两类，坐标型动态规划和序列型动态规划。前者每次考虑的是一个坐标点，而后者考虑的是某个点以前的一段序列。坐标型动态规划起始i = 0，序列型动态规划起始i = 1。

序列型动态规划

序列型动态规划经常要人为添加第“0”个状态。不过是否添加还是要根据状态和状态转移函数进行决定。不能生搬硬套公式。

动态规划的实现方式

循环（从小到大递推）

后面的状态只与前面的某几个状态有关，可以用前面的某几个状态推导后一个状态。

记忆化搜索（从大到小搜索）

动态规划的本质是解决了重复计算的搜索。使用通常的深度优先搜索可能会常常会出现一些点被重复搜索的情况。动态规划记录了被搜索过的点的结果，再后续的搜索中就可以直接利用这些结果而不再进行重复搜索。

当动态规划遭遇 1. 状态转移函数过于复杂，不是顺序性，或者 2. 难以找到初始状态的情况，就可以使用记忆化搜索。
