Hadoop 3 与 Hadoop 2 以及 Spark 的对比

Posted on 2019-11-26 Edited on 2019-11-30 In 学习笔记

翻译自 Igor Bobriakov 的文章，Hadoop 3: Comparison with Hadoop 2 and Spark。一切权利归原作者所有。

大数据生态系统内工具种类繁多，刚接触时很容易陷入迷茫。这篇文章介绍了大数据生态内的两大重要框架：Hadoop 和 Spark，相信可以给初学者带来一些帮助。在下也是初学者中的一员，有很多概念和术语都没有完全搞清。如有疏漏，敬请见谅指正。

我将以注释的形式为原文添加补充信息

以下是正文：

2017 年 12 月发布的 Hadoop 3 标志着数据科学新时代的来临。Hadoop 框架是整个 Hadoop 生态的核心，很多库的运行都倚赖于它。

本文将介绍与 Hadoop 2 相比，Hadoop 3 最大的变化。本文还将介绍 Hadoop 与 Apache Spark 的区别，也会为你的项目提供一些工具选择上的建议。

综述

Hadoop 2 和 Hadoop 3 是基于 Java 开发的数据处理引擎，分别于 2013 和 2017 年发布。Hadoop 的诞生最初是为了处理硬盘上的数据分析，属于批处理任务的范畴。因此最初的 Hadoop 并不支持实时数据分析和交互式任务。

Spark 2.X 发布于 2016 年，是基于 Scala 开发的数据处理和分析引擎。彼时许多大型互联网服务都十分依赖于即时数据处理，因此对于数据的实时分析能力变得至关重要。Apache Spark 便是为实时数据处理而设计的。它因能够高效处理数据流，还支持交互式工作模式而变得流行。

Hadoop 和 Spark 都是 Apache 2.0 协议下的开源软件。

抽象程度、学习和使用难度

这些工具的抽象程度有很大区别。具体来讲，Hadoop 的抽象程度更低，而 Spark 更高。学习和使用 Hadoop 会更有难度，因为开发者必须掌握许多底层的操作。同时，Hadoop 仅仅是系统中的核心引擎，要使用更高级的功能往往还要倚赖其它组件。这样的结构使得系统的复杂性大大提升。

于 Hadoop 不同，Apache Spark 提供了一整套的数据分析工具。Spark 的核心概念是 RDD（Resilient Distributed Dataset，弹性分布式数据集）。它内建有许多高级方法来处理 RDD。它还提供许多能在集群中发挥作用的库。例如，MLlib支持机器学习，Spark SQL支持 SQL 查询等等。

硬件成本

Hadoop 主要运行在硬盘上，所以它的运行不需要很多内存，这比使用大内存要更便宜。Hadoop 3 消耗的硬盘空间比 Hadoop 2 要小，这是由于其在容错系统方面的改进。

Spark 在内存模式下消耗的内存要远大于 Hadoop，这使得它的硬件成本更高。

编程语言

两种 Hadoop 都通过 Hadoop Streaming 支持多种编程语言，但是用的最多的还是 Java。Spark 2.X 支持 Scala、Java、Python 和 R。

运行速度

通常情况下 Hadoop 比 Spark 慢，因为它运行在硬盘上。Hadoop 不能将数据缓存到内存中。Hadoop 3 的运行速度比 Hadoop 2 快了 30%。这是因为它在 MapReduce 的基础上引入了原生 Java 搭建的 Map output collector。

在内存上，Spark 的信息处理速度是 Hadoop 的 100 倍。在硬盘上，这个数字是 10 倍。

安全性

Hadoop 被认为是比 Spark 更安全的，因为它使用了 Kerberos（computer network authentication protocol），还支持 ACL（Access Control Lists）。Spark 仅提供了基于 shared secret 的密码保护。

容错性

Hadoop 2 通过冗余的方式提供容错性。其储存的每一块数据都被复制到另外两个备份中。这意味着，Hadoop 2 需要三倍于文件大小的存储空间。这对于硬盘容量是一种很大的浪费。

Hadoop 3 的容错性则是通过纠删码技术实现的。这种技术通过另一块数据和奇偶校验块进行数据恢复。每有两个数据块，Hadoop 就创建一个奇偶校验块。与 Hadoop 2 的三倍存储空间消耗相比，Hadoop 3 只消耗了 1.5 倍。Hadoop 3 的容错能力和 Hadoop 2 是一样的，但是更加节省空间。

Spark 通过重新计算 DAG（Directed Acyclic Graph，有向五环图）进行数据恢复。DAG 由点和线构成。点代表 RDD，线代表对 RDD 执行的操作。如果有一部分数据发生了丢失，Spark 可以通过对 RDD 重新应用 DAG 中对操作来恢复数据。需要注意的是，每次重新计算 RDD 都需要等待 Spark 完成所有必要计算。Spark 还通过检查点的方式避免错误。

YARN 版本

Hadoop 2 使用 YARN 1. YARN（Yet Another Resource Negotiator）是一种资源调度引擎。它能够管理系统中的可用资源（CPU、内存、硬盘）。同时，YARN 还能执行任务调度。

在 Hadoop 3 中 YARN 被升级到了版本 2。新版本相比旧版带来了可用性和扩展性方面的重大改变。
